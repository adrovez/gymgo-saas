Patrón Arquitectónico

La app sigue una arquitectura Feature-Based con separación estricta entre infraestructura reutilizable (core/) y módulos de negocio (features/). Todos los componentes son standalone (sin NgModules). El estado se maneja con Angular Signals en lugar de NgRx para mantener la simplicidad.

Principio clave: core/ no importa nada de features/. Los features pueden importar de core pero no entre sí.

Flujo de Autenticación

Login

1. Usuario ingresa email + password + tenantId → 2. POST /api/v1/auth/login → 3. tap(): StorageService.saveSession() → 4. _session signal actualizado → 5. router.navigate(['/app/dashboard'])

Request autenticado

Componente hace HTTP call → authInterceptor clona req → Agrega Authorization + X-Tenant-Id → API valida y responde

Guard de rutas protegidas

Navega a /app/* → authGuard: isAuthenticated()? → ✓ permite acceso | ✗ redirect /login

Interceptor HTTP — Detalle

export const authInterceptor: HttpInterceptorFn = (req, next) => {
  const storage = inject(StorageService);
  const token    = storage.getToken();
  const tenantId = storage.getTenantId();

  if (!token) return next(req);

  let headers = req.headers.set('Authorization', `Bearer ${token}`);
  if (tenantId) headers = headers.set('X-Tenant-Id', tenantId);

  return next(req.clone({ headers }));
};
